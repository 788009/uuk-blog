> 提到的事实存在诸多错误，但本文的目的是记录思考本身。

现代图形用户界面（Graphical User Interface，GUI）框架可以分为两种模式：即时模式（Immediate Mode，如游戏）和保留模式（Retained Mode，如微信）。研究后者时，不难发现，不论是基于 Web 的 WebUI、Electron、Tauri、PyWebView，还是基于其他绘图引擎的 WinUI、Swift UI、Qt、Flutter、React Native，无一例外采用树状结构来布局和管理 UI，比如 HTML 的嵌套式标签。一个事物能成为主流，必然有其物质基础与推动力量。在二十世纪八十年代 Smalltalk-80 的 MVC (Model-View-Controller) 模式首次使用基于层级的窗口管理之前，程序员只能基于坐标来管理 UI，比如在某点画一个按钮，这在当时的条件下是最优解，因为当时显存只有 4-16 KB，CPU 计算能力仅为现代手机的数千分之一，还无法支持节点坐标改变的递归计算，以及计算机才刚刚向个人普及，应用场景十分简单，没有复杂的需求。随着需求的日益复杂，人们发现基于坐标管理 UI 认知负荷太大，加上硬件性能提升，以及 Smalltalk 引入了面向对象思想（组件树本质就是对象），MVC 模式应运而生，其思想一直延续到现在。

上述过程本质上可以归结为物质基础的进步和矛盾对事物发展的推动作用。具体到 GUI 框架的语境来说，物质基础是硬件性能与其他思想和技术的发展，而矛盾则是人们增长的需求于现有技术不足的矛盾。使用这个基本模型，可以推导整个 GUI 框架的演进史：互联网的兴起极大地加剧了用户界面的复杂化，人们发现手动操作组件树（比如网页的 DOM）的认知负荷逐渐逼近承受上限，于是开始寻找解决途径，其中典型代表有 2013 年的 React 和 2014 年的 Vue。这两个框架的实现原理不同，目的却殊途同归——都实现了状态与表现的解耦。这种数据驱动的模式让人们从同时管理数据与 UI 的同步地狱中解放出来，把 DOM 元素交给框架管理，自己则能够更加专注于业务逻辑，使效率大幅提升。但是 React 和 Vue 也带来了可观的性能代价，尤其是在几年后互联网开始向发展中国家渗透，低端手机的普及使得优化网页性能的需求日益增加，这就催生了编译时预处理的 Svelte 和调度时分片的 React Fiber，二者都诞生于 2016 年。不只是网页，桌面与移动端应用的跨平台需求也促使人们积极探索新的方案，比如 GitHub 在 2013 年推出的 Electron 和 Google 在 2015 年提出的 Flutter 都是极为优秀的实践。

我们亦可以畅想未来，比如将来 VR、AR 和全息投影技术的发展与普及，可能会催生新的 UI 布局模式：人的视网膜是二维的，因此只能一次获取二维平面的所有信息，但无法一次获取三维空间的所有信息，比如在不同视角下，两个元素的相对关系可能完全不一样，这使得传统的二维嵌套布局失效，UI 需要从“容器包含”逻辑转向“空间位置”逻辑。除此之外，AI 技术的发展可能彻底重塑交互模式，比如从点击变成自然语言，这是更加根本的变革，使得 UI 布局与管理本身变得不再必要，配合更加先进的硬件性能，人们只需要直接描述自己的需求，AI 就可以即时生成出最合适的 UI 界面。再进一步，脑机接口甚至可以让“用户界面”这个需求都消失殆尽，这又是更加底层的变革了。

回顾异步编程的历史，我们不难发现惊人的相似性。人的本能是异步的，比如可以边做菜边聊天，但 CPU 不是，CPU 只能顺序执行指令，因此如何在同步的 CPU 上满足异步需求，就成了很自然的问题。从二十世纪六十年代开始，人们就引入了进程的概念。那个时候，程序员还是手动管理进程，有需要就 fork 一个新进程，这确实是一种有效的解决方案，但也原始又笨重，多进程就像是同时开若干个工厂，但切换进程时，必须执行上下文切换，把上一个工厂整个存入内存，之后回来时再重新开启，资源消耗巨大，同时手动管理也十分麻烦。九十年代，多线程出现，CPU 切换线程时只需要把寄存器快照和内核栈信息存到内存，不同线程在同一个进程中共享内存，使得任务切换更加轻便。同时，回调思想产生，人们不需要手动管理线程，只需要告诉 CPU 执行完某个任务就调用指定函数，这有效减轻了认知负荷。线程的出现也顺应了 GUI 与互联网的兴起（C10K 问题）：用户点击按钮之后，界面不能卡死；CPU 时间非常宝贵，等待网络请求响应的时间可以执行数亿条指令，这是不能浪费的。但是回调的非线性逻辑十分反直觉，随着互联网的进一步发展，业务逻辑愈发复杂，回调层数越来越多，这侵蚀着人们的心智，人们逐渐陷入回调地狱，因此 Promise 出现了；同时大量的并发需求（C10M 问题）导致多线程也太重了，加上复杂的锁竞争和竞态条件，人们开始使用协程。协程其实出现得比线程还要早，但当时还没有这么高的需求，实际上在 2010 年之后才开始流行。协程实现了纯用户态的多任务，比线程切换轻得多，在 I/O 密集型场景下足够支持百万级并发。Promise 本身也是非线性的，对其的改进与协程结合，最终产生了 async / await，这本质上是非阻塞异步的同步化表达，允许人们把异步当成同步来写，认知负荷极小；虽然语法糖必然带来一些性能损失，但此时硬件的进步已足够支持。

GUI 框架的发展与异步编程的发展，关键推动力之一是人有限的注意力与无限增长的业务逻辑复杂度之间的矛盾，而解决矛盾的措施本质上都在做同一件事：抽象。抽象的目的在于减少复杂度，比如将坐标抽象为组件树的元素，人们只需要关心元素的嵌套层级，而无序关心具体的坐标数字；Flutter 将不同操作系统的底层接口抽象成了平台无关的统一接口，使得程序员（在大部分时候）不需要关心面向的平台，而只需要面向 Flutter 提供的接口编程；异步也是一样，复杂的回调关系被抽象成 Promise，再抽象成 async / await。其实抽象在生活中也很常见，最典型的例子是汽车，汽车上的每一个组件，比如油门、方向盘，都是抽象，使得人们无徐关心具体的机械结构，只需要学习如何踩油门、如何操作方向盘。抽象还提供了扩展性，如果将来想让机器人学会开车，那么只需要让机器人学会操作汽车上的组件，对接抽象好的接口即可，而不需要直接操作底层零件。

物质基础的进步带来需求与矛盾，人们用抽象来解决，而抽象本身是一种进步，有时候也会带来新的矛盾，于是再用抽象来解决——事物就在这个过程中不断进步。

若将思考放到更高维度，可以发现，基于物质基础与矛盾运动的辩证法是非常精妙的。我们不妨用这个模型来研究辩证法本身：辩证法出现于工业革命之后，而在此之前，占据主导地位的是形而上学，这或许在某种程度上映射了农业社会的稳定性，几千年来，人们的生产与生活几乎没有发生过本质的变化，只是在一个个春夏秋冬的循环中机械地度过时间，加上牛顿力学的成功，人们可以用一套精确的数学方程来解释世界，在这样的条件下，人们非常自然地认为世界是孤立、静止与机械的，形而上学也确实有其普适性。然而，工业革命彻底打破了这个格局，它将孤立变成了普遍联系、将静止变成了动态转化、将机械叠加变成了矛盾运动，形而上学普适性难以维持，人们自然转向了其他架构的模型。

这与机器学习何其相似！如果我们把人类的认知过程抽象来看，似乎也是在做同一件事情：拟合。在静止的农业社会，人们尝试拟合世界的运行规律，于是诞生了同样静止的形而上学，其泛化能力确实足以解释当时人们能遇到的事物，性能让人满意。然而，工业革命使得社会运行的数据分布发生了剧烈偏移。基于农业社会数据训练的模型，其泛化能力必然大打折扣；而采用新数据拟合的辩证法，自然轻松超越了过时的形而上学。

这自然引出了新的问题：世界的运行数据还会再次发生剧烈偏移吗？虽然辩证法强调“永恒发展”与“普遍联系”，似乎本身可以跟着社会的演化而演化，却也有一些基本假设，但将来有没有可能出现翻天覆地的变化，直接推翻了这些基本假设，使得辩证法无法拟合这些复杂的规律？完美模型是否存在？

学习到世间所有规律的完美模型似乎是存在的，只要给这个模型输入足够的数据，就能得到准确的输出。比如问这个模型“明天是否下雨？”，只要给模型准确的大气数据，模型就可以输出“明天下雨”或者“明天不下雨”。

这是一个很迷人的思维模式，不过这不禁让人联想到决定论与拉普拉斯妖，它将世界看作函数 $y = f(x)$，认为一切事物都是在一开始就确定的。缺陷也很明显，$f(x)$ 由两个部分构成，映射关系 f 与自变量 x，先说自变量 x，我们没有获得整个宇宙的所有数据，就算是“获得准确的大气数据”都做不到，这直接从表面上扼杀了决定论的活力。此外，由于完美模型考虑了所有变量，而变量之间会互相影响，这就产生了悖论：预测结果影响了决策，导致变量本身发生变化，刚刚的预测也就失去了价值。如果模型说明天要下雨，因此我打算明天不出门，如果因此（比如蝴蝶效应）导致明天不下雨了，完美模型变得不完美。

而本质的问题在于映射关系 $f$，即，世界是简单的线性映射吗？实际上，现代量子力学告诉我们，微观层面的 $y = f(x)$ 输出的不再是一个确定的值，而是一个波函数；只有在观测的瞬间，它才坍缩成我们看到的现实。也就是说，世界在微观层面就是随机的，而不是确定的，永远无法通过前一个状态准确预测后一个状态，只有观测才能得到确定，否则只能得到概率分布，这也导致就算真的存在所谓完美模型，也只有后验价值，而没有先验价值。

为什么微观是随机的，宏观却是有序的呢？这可以用大数定律来解释，即足够多的随机事件会让总体趋于确定，微小的偏差并不会影响整体的结果。尽管微观粒子是随机测不准的，但在宏观尺度上，海量的微观粒子相互作用，统计规律开始主导一切，确定性的秩序从而涌现出来。比如抛很多次硬币，抛到正面的次数一定会稳定在一半左右，具体多一次少一次都不会改变这个事实。这又是一个十分迷人的模型，大数定律允许了随机性，又得到了确定性，不仅可以在物理学解释微观粒子的无序与宏观事物有序，还可以解释历史趋势的必然性，因为历史趋势就是大量看似随机的事件产生的确定性，比如就算 Smalltalk-80 没有在 1980 年发布，基于树形结构的 UI 布局与管理模式依然会在差不多的时间出现；就算 Promise 不叫 Promise，回调地狱也会让另一个名字的抽象模式出现；就算马克思没有出生，辩证法也一定会在当时的社会背景下以其他方式出现。这就是历史的必然，也允许了必然趋势下的自由，比如，午饭吃什么。

这么看来，大数定律似乎可以被认为是世界的底层逻辑了。然而，大数定律依然不完美，根本原因在于它虽然假设世界是概率的，但同时假设不同事物的概率是独立同分布（Independent and Identically Distributed，IID）的，可惜现实并非如此，现实世界是复杂的，非独立同分布（Non-IID）的，事物之间存在复杂的关联，这也是为什么会出现黑天鹅，也就是极其罕见、影响巨大的事件，比如经济危机，一个事件会彻底改变原本平稳的均值，这是大数定律完全无法解释的。

如此美妙的模型就这么破碎了，多么可惜！自然地，有没有更普适的模型呢？其实谜底就在谜面里：虽然世界是非独立同分布的，但这个分布本身是确定的，从微观上来说，就是粒子下一个状态的概率分布是确定的，这不就是足以兼容目前所有讨论的模型吗？

但是还有一个疑问：如果确定演化的概率分布就是世界的底层逻辑，那为什么黑天鹅是罕见事件而不是家常便饭呢？这是因为，抑制黑天鹅的负反馈是普遍的，比如摩擦力、能量耗散、生态平衡、社会契约，这些力量一直在试图将偏离均值的波动拉回来；而在那些正反馈占主导的罕见时刻，非独立同分布的关联性会被瞬间放大，宏观的“黑天鹅”便诞生了。这也是为什么大数定律就像牛顿运动定律一样，在日常生活中确实够用了。

至此，我们打破了决定论，一路走到了 Non-IID，把确定的结果变成了确定的概率分布——但，这不还是决定论吗？世界其实是早就定好赔率表的赌博，一个完美的统计宇宙完全可以是一个没有灵魂的自动化工厂，我们将要做的每一件事都会符合某种确定的概率分布，这真的自由吗？也许我们的努力、我们的奇思妙想都不过是统计均值周围的一个噪点而已。这只是把 $y = f(x)$ 变成了 $P(Y) = f(x)$ 而已，到头来，我们还是无力的提线木偶吗？

其实，出路是有的。我们解决了 $f(x)$ 中 $f$ 的问题，但还没有解决 $x$ 的问题，我们根本无法获得所有自变量，当然也无法根据模型进行推演，概率分布模型依然只有后验价值，没有先验价值。这在无力的人类看来，和完全随机有什么区别呢？没有区别。尽管我们做的每一件事也许都可以用概率分布模型来解释，但没有人，现在没有，将来也不会有，可以预测我们的下一步，因为不可能在宇宙中，模拟整个宇宙。也许确实有什么统治一切的规律在约束我们，但我们为什么要在意呢？我认为，这就是出路。

结果，一路思考下来，得到的结论是没必要思考，那思考还有什么用？答案很简单，这一切都是有意义的，不思考这些也到不了“不必在意”的这一步，也不会经历这愉快的、令人心旷神怡的体验了。
